# Echarts 图表

这段时间一直在记账，用于严格控制自己消费，在公司经常能看到他们做项目用到 echart 图表，今天突发奇想，为什么我不用 echart 来把我的账单可视化呢，说干就干，于是就有了这个折线图，我觉得很好看，毕竟花了点心思在样式上面，搭配了主题颜色，虽然代码比较多，但好看就完事了！还有许多自带的功能，什么切换柱形图，放大，保存图片之类的，还是挺有意思，一想到这么有意思的东西以后要拿来做项目我就哭哭，通过这个折线图可以看出 7 月份消费很高，至少对我来说，主要原因是来新城市，有许多要买的东西，还有一个是入职体检很多钱，也买了几件衣服，等八月份过后可以出个消费对比图，应该就能看到我的正常消费水平了。

什么？？你说你看不到我的折线图？？点击下面的按钮刷新一下吧～

看不到的原因应该就是页面还没有加载出来就开始找 echart 盒子，解决办法应该是添加定时器之类的吧，如果是 vue 可以放在 mounted 等待 dom 元素加载完成后执行函数，但我实在是懒得写了，就用重新加载来代替吧，毕竟那个按钮很好看不是吗，还会像心跳一样蹦跶，哈哈哈，可爱捏。

还有一件值得说的事情是，我把博客的搜索功能给解决了，之前只能搜索在 22 年的博客，23 年的博客搜索不到，原因是配置出现了问题，需要在系统变量中以键值对的形式配对 algolia 秘钥，然后每次上传博客之前都需要 hexo algolia 更新搜索。要说修复没用那是不应该的，虽然没什么人知道我的博客，就算知道也不会特地去搜索，但主要是方便我来查找相关代码，或者回忆，好比给我的记忆宫殿添加了一个搜索功能，还是很棒的，我是这样安慰自己的 QWQ。
